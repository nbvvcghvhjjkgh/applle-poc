[ DOM INVADER — DEFINITIVE CONTEXT TEST ]

This test PROVES where code executes: Page vs DevTools Extension Context

HOW THIS WORKS:
This page triggers a real innerHTML sink so DOM Invader detects it.
The payload sends a webhook request with the exact execution context (URL, protocol, chrome APIs, document.title).

KEY: If you receive TWO separate webhook requests:
• Request #1: from https://your-page-url (page context — this is expected/normal)
• Request #2: from chrome-extension:// or devtools:// (extension context — THIS IS THE VULNERABILITY)

If only ONE request arrives (from the page) → No vulnerability.
STEPS:
1. Make sure DOM Invader is ON + Augmented DOM is ON
2. Click "TRIGGER SINK" below
3. Open DevTools → DOM Invader tab → you should see a Sink entry
4. Press Esc for Console
5. Click the Stack Trace link in the sinks table
6. Check webhook — count how many requests you received
